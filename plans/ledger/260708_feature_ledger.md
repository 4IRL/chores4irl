## Settings / device-control panel (added 2026-06-28)

The rotate-screen control will live inside a new collapsible **settings panel** rather than as a bare NavBar button (see `plans/feature/rotate-screen-button/`). The panel is a single NavBar toggle (a gear/settings icon that swaps to an 'X' when open, co-located at the upper-right-hand corner) which expands into a **single-row overlaid banner of kiosk/device control icons**. The rotate control is wired and functional; the controls below ship initially as **disconnected, optimistic icon placeholders** (rendered, labelled, inert — no backend) for future work to connect.

- [ ] **Brightness control** — adjust the kiosk display backlight from the settings panel. Likely a host-bridge mirroring rotate (frontend → backend writes a brightness state file → host agent applies it via the backlight sysfs interface or `ddcutil` for the HDMI panel). Placeholder icon (`Sun`) ships first, disconnected.
- [ ] **Screen blank / wake toggle** — manually blank (sleep) or wake the display on demand from the panel, complementing the auto screen-blank-disable already configured on the Pi. Placeholder icon (`MonitorOff`) ships first, disconnected.
- [ ] **Auto screen blank / wake** — allow the user to set the screen sleep and wake times (instantiated as 9 pm and 6 am, respectively, per F1)
- [ ] **Restart** — trigger a controlled restart from the panel — of the app stack (`docker compose restart`) and/or the Pi host — behind a confirmation dialog. Likely another host-bridge action. Placeholder icon (`Power`) ships first, disconnected.
- [ ] **Undo** — allow the user to recover from accidental touch and chore completion. We should consider how big of an undo cache to practically allow. If complexity or cache size limited, 1-2 undos should be considered sufficient.
- [ ] **Redo** — re-apply an action undone via the undo feature; pairs with the undo cache above. Same bounded cache consideration (1–2 steps if complexity/size limited). Placeholder icon (`Redo2`) ships first, disconnected; undo (`Undo2`) likewise relocates into this panel.
- [ ] **Rotate** — rotate screen button on the UI to dynamically allow the user to change the touch behavior and display orientation. This way, regardless of how the screen is mounted, the user can adjust, rather than requiring a developer to adjust the screen
